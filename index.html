Introduction

<

This is an app that allows users to create routes, save routes and try new routes. There are similar works like Mapmyrun, but none of them recommend routes personally for users.

Up to now, the routes recommendation is based on personal information such as bmi and active level, and some features of routes such as the distance between current location of the user and the start location of this route, route length and difficulty level (flat or hilly or curved).

Future works include 1) updating the active level of user based on the usage of this app 2) some user profile management like changing password, username etc. 3) "like" others routes 4) ...

Sign up

A new user needs to sign up with an email address and a password. The system will try to search for user by email to check whether this is an old user. If the email is not found, an alert window will pop up to ask whether user want to sign up. Click yes to continue.

Users need to follow the instructions and fill in all blanks. All the requring information is basic. The active level of user is a measurement of how active the user is, which based on subjective jugdement.

Sign in

After successfully signing up, users can login by email and password.

Show routes around

After login, the system will show all the routes that is around users. Users can see the details of each route by clicking directly on each polyline, or select the number of route throught the spinner.

Create a route

Users can create a route by clicking on the button above. It will lead user to another activity. In the new activity, user can start when they are ready and stop after completing the route they want to create. Then, the system will pop up a window asking whether user want to save this route. If yes, user needs to input more infomation about this route, such as give it a name, describe the features of this route and choose whether to share it to others.

Show my routes

If user want to see his/her own routes, they can click on the menu bar and choose the "My Routes". This will take a little bit while as system is reading from MySQL. (I forgot to add a progress bar here. I will add it afterwards. :)

Recommend Routes

Test Case 1

The recommendation is simmple at this stage. The basic idea is : users BMI and active level decide the disatance, the difficult level of routes that would be remmended to them. And overall, all the recommended routes will be within 5 km from users current position

(There are 3 routes in total, two of them are level 2 and 1 is level 1. The routes are short because for now, the system relys on mhealthplay databse which can only be connted via NU network. I tried several times to record a longer route, but once the network is unstable the upload will fail. So there are some limitations of the length, but we check the system by the different difficult level of routes.)

The first test case is for a person whose BMI is in the healthy range and active level 2. She will be recommended routes shorter than 5 km and difficult level is 1 or 2.

Recommend Routes

Test Case 2

The second test case is a person whose BMI is larger than 25, which is the upper bound of healthy range and active level is 0. He will be recommended routes shorter than 3 km and difficuly level is 1.

Try a route

Select and Direct to the route

Users can choose to try any routes that is recommended for them. After clicking on a route, there will be a button shown to users. If they are ready to start, they can click it. Here, "ready" means they are near the starting location of this route. There are two icons at the right bottom which will open the google map to help user get the location.
